<Fill Documentation Strategy>
Inventory the project's documentation and define the maintenance process:
- What documentation artifacts exist (READMEs, requirement docs, API docs, architecture docs, changelogs)?
- Where does each artifact live and what does it cover?
- Which docs must be updated when code changes? (e.g. README for new CLI flags, requirements doc for new deps)
- What is the process for keeping docs in sync with code?
- Are there generated docs (e.g. Sphinx, typedoc, rustdoc)? How are they built?

Ask the user:
- Are there docs outside the repo (wiki, Confluence, Notion, Google Docs)? If so, where?
- Which docs are user-facing vs. developer-facing?
- Is there a documentation review process (e.g. docs required in PRs)?
</Fill Documentation Strategy>

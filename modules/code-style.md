## Code Style

- Follow the existing code style and patterns in the project
- Don't add comments for self-explanatory code
- Don't add type annotations or comments to code you didn't change unless required by the Documentation module
- Keep functions small and focused
- Prefer clarity over cleverness
- If a formatter or linter is configured, run it before considering a change complete

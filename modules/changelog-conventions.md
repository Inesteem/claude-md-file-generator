## Changelog Conventions

- Maintain a CHANGELOG.md following the Keep a Changelog format (Added, Changed, Deprecated, Removed, Fixed, Security)
- Use semantic versioning: MAJOR for breaking changes, MINOR for new features, PATCH for bug fixes
- Write changelog entries from the user's perspective — describe the impact, not the implementation
- Add changelog entries as part of the PR, not as an afterthought at release time
- Never delete old entries — the changelog is an append-only history
